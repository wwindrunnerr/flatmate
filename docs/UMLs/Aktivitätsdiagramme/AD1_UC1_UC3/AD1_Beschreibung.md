Das Aktivitätsdiagramm fasst die drei zusammengehörigen Anwendungsfälle UC1 „WG anlegen“, UC2 „Einladung erzeugen“ und UC3 „WG beitreten“ zu einem durchgehenden Gesamtprozess zusammen. Es zeigt, wie zunächst ein eingeloggter Benutzer eine neue WG erstellt und dabei automatisch die Admin-Rolle erhält. Anschließend kann dieser Admin weitere Personen über einen Einladungslink oder einen Einladungscode in die WG einladen. Im letzten Teil des Ablaufs tritt ein eingeladener Bewohner der WG bei, indem der erhaltene Link geöffnet oder der Code eingegeben wird. Das System prüft dabei jeweils die Gültigkeit der Eingaben und reagiert bei Fehlerfällen, zum Beispiel bei einem ungültigen WG-Namen, einem erreichten Einladungslimit oder einem abgelaufenen beziehungsweise ungültigen Einladungscode, mit einer Fehlermeldung. Damit verdeutlicht das Diagramm den fachlichen Zusammenhang der drei Use Cases: UC1 schafft die WG, UC2 ermöglicht die Einladung weiterer Mitglieder und UC3 schließt den Prozess durch den Beitritt eines neuen Bewohners ab. Dieser Zusammenhang entspricht auch der funktionalen Anforderung F05 zur WG- und Benutzerverwaltung im SRS.

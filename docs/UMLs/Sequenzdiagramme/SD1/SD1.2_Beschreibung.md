Das Sequenzdiagramm zeigt die zeitliche und technische Abfolge der drei zusammenhängenden Anwendungsfälle „WG anlegen“, „Einladung erzeugen“ und „WG beitreten“. Dabei wird dargestellt, wie die beteiligten Akteure über Browser, Server und Datenbank miteinander interagieren. Zunächst legt der Admin eine neue WG an, wobei der Server die Eingabe validiert, die WG in der Datenbank speichert und den Ersteller als Admin-Mitglied einträgt. Anschließend kann der Admin eine Einladung erzeugen; das System prüft dabei Rechte und Limits, generiert einen Einladungscode mit Ablaufdatum und speichert diesen in der Datenbank. Im letzten Schritt verwendet der eingeladene Bewohner den Code zum Beitritt, woraufhin das System die Einladung überprüft und bei gültigem Ergebnis die Mitgliedschaft als Bewohner anlegt und den Vorgang protokolliert. Das Diagramm verdeutlicht damit, wie die drei Use Cases technisch aufeinander aufbauen und gemeinsam den vollständigen Aufnahmeprozess neuer WG-Mitglieder innerhalb der funktionalen Anforderung F05 abbilden.

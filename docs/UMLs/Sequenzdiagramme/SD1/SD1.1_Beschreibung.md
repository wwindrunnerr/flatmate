Das Sequenzdiagramm zeigt die zeitliche Interaktion zwischen den beteiligten Akteuren und Systemkomponenten bei den drei zusammenhängenden Anwendungsfällen „WG anlegen“, „Einladung erzeugen“ und „WG beitreten“. Zunächst sendet der Admin über den Browser die Daten zur Erstellung einer WG an den Server, der die Eingabe prüft, die WG in der Datenbank speichert und die Admin-Mitgliedschaft anlegt. Anschließend kann der Admin eine Einladung erzeugen; dabei prüft das System die Berechtigungen und das Einladungslimit, erstellt einen Einladungslink und speichert ihn. Im letzten Abschnitt nutzt der eingeladene Bewohner den Link bzw. Code, woraufhin der Server die Einladung überprüft und bei erfolgreicher Validierung die Mitgliedschaft des neuen Bewohners in der Datenbank anlegt. Das Diagramm verdeutlicht damit nicht nur die Reihenfolge der Nachrichten zwischen Browser, Server und Datenbank, sondern auch den fachlichen Zusammenhang der drei Use Cases: Die WG wird zuerst erstellt, danach wird eine Einladung generiert und schließlich tritt ein neuer Bewohner der WG bei. Dieser Ablauf gehört zur funktionalen Anforderung F05 der WG- und Benutzerverwaltung.

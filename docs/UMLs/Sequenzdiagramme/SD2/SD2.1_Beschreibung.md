Das Sequenzdiagramm beschreibt den zeitlichen und technischen Ablauf der zusammenhängenden Anwendungsfälle „Ausgabe erfassen“, „Salden anzeigen“ und „Ausgleich markieren“ innerhalb der Budgetverwaltung. Zunächst erfasst ein Bewohner über den Browser eine neue Ausgabe und übermittelt die zugehörigen Daten an das System. Der Server prüft die Eingaben, speichert die Ausgabe, verarbeitet optional ein Belegfoto und berechnet je nach gewählter Methode die Anteile für die beteiligten Mitglieder. Anschließend können die Bewohner die Saldenansicht öffnen, woraufhin das System die relevanten Daten ausliest, die aktuellen Salden berechnet und zusätzlich Ausgleichsvorschläge anzeigt. Wird eine Zahlung als erledigt markiert, prüft das System die Angaben, speichert die Zahlung und berechnet die Salden erneut. Das Diagramm verdeutlicht damit, dass die Anwendungsfälle fachlich und technisch direkt zusammenhängen: Die erfassten Ausgaben bilden die Grundlage für die Berechnung der Salden, und der markierte Ausgleich aktualisiert diese Werte erneut. Dieser Gesamtprozess gehört zur funktionalen Anforderung F03 Budgetverwaltung im SRS.

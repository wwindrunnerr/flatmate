Das Sequenzdiagramm beschreibt den technischen Ablauf der zusammenhängenden Anwendungsfälle „Transaktion erfassen“, „Salden anzeigen“ und „Ausgleich markieren“ innerhalb der Budgetverwaltung. Zunächst erfasst ein Bewohner eine neue Transaktion über den Browser und übermittelt die eingegebenen Daten an den Server. Das System validiert die Angaben, verarbeitet optional ein Belegfoto, speichert die Transaktion in der Datenbank und berechnet anschließend die Anteile der beteiligten Mitglieder. Danach können über die Saldenansicht die gespeicherten Transaktionen, Mitglieder und Anteile ausgelesen werden, sodass das System die aktuellen Salden und passende Ausgleichsvorschläge berechnen und anzeigen kann. Wird anschließend ein Ausgleich als erledigt markiert, prüft der Server die Plausibilität der Zahlungsdaten, speichert die Zahlung und berechnet die Salden erneut. Das Diagramm verdeutlicht damit, dass diese drei Anwendungsfälle technisch direkt aufeinander aufbauen: Die erfassten Transaktionen bilden die Grundlage für die Saldenberechnung, und ein markierter Ausgleich aktualisiert diesen Zustand wieder. Dieser Gesamtprozess gehört zur funktionalen Anforderung F03 Budgetverwaltung im SRS.
